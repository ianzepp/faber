# Faber v0.35.0

This minor release continues the post-split Faber/Radix line with a broader
language-surface cleanup, package-tool improvements, and standard-library
runtime groundwork.

## Highlights

- Adds `faber test` support backed by the generated Rust test harness.
- Adds and expands `faber explain` coverage for active glyphs, keywords, and
  legacy redirects.
- Moves the explain corpus to TOML front matter.
- Removes legacy ASCII and Latin alias spellings for glyph-first source forms.
- Introduces `sponte`, `fixus`, and `T ∪ nihil` syntax for declaration
  optionality and nullable value domains.
- Starts target-neutral `norma` library import resolution and JSON/TOML runtime
  value groundwork.
- Tightens parser, semantic, codegen, example, and documentation alignment after
  the v0.34 Faber/Radix split.

## Notes

- `faber` remains the user-facing project and package tool.
- `radix` remains the compiler-developer inspection tool.
- Compatibility aliases on `faber` for compiler inspection commands remain
  available in this release.
