# Faber v0.36.0

This minor release advances the post-split Faber/Radix line with broader
backend coverage, MIR and Wasm progress, host syscall groundwork, and a
source-surface cleanup for range iteration.

## Highlights

- Renames range loops from `itera pro` to `itera ab` with a hard pre-release
  cut and updated examples, docs, parser diagnostics, explain entries, MIR, and
  codegen.
- Expands MIR lowering and validation across control flow, aggregates,
  optionals, runtime intrinsics, diagnostics, arrays, ranges, and Wasm text
  emission.
- Adds and gates Go, TypeScript, and Wasm exemplar baselines.
- Splits the Faber and Radix command implementations into focused modules.
- Improves Rust backend coverage for optional values, dynamic values,
  collections, enums, cursor functions, methods, and CLI codegen.
- Adds macOS host syscall and Wasm/component routing proofs for the emerging
  host runtime boundary.

## Notes

- `faber` remains the user-facing project and package tool.
- `radix` remains the compiler-developer inspection tool.
- `itera pro` is no longer accepted as a range-loop spelling; use `itera ab`.
- Tag range: `v0.35.0..v0.36.0`.
